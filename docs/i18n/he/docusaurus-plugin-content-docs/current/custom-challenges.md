# אתגרי אנטי-ספאם מותאמים אישית

Bitsocial לא מניח הגנת דואר זבל אוניברסלית אחת. כל קהילה יכולה להחליט מה דורש פוסט או תשובה חוקיים, והאתגר הזה יכול להיות קל משקל או דעתני כפי שהמפעיל רוצה.

לזרימה המלאה באנגלית פשוטה מכתובות קהילתיות ועד לפרסום מותאם אתגרים, קרא [הסבר שלם של הדיוט על פרוטוקול Bitsocial](./layman-protocol-explanation.md).

## איזה אתגר יכול להיות

- קפצ'ה
- בדיקת גיל חשבון או מוניטין
- אימות SMS
- תשלום קטן
- דרישת אסימון או NFT
- כללים מבוססי IP
- רשימת היתרים
- כל מדיניות אחרת שניתן לבטא בקוד

## למה זה חשוב

ההתנגדות לספאם נשארת מקומית לקהילה במקום להיות מיקור חוץ לרשות ניהול פרוטוקול. הצומת המארח את הקהילה יכול לתקשר את האתגר עמית לעמית, מה שאומר שקהילות יכולות להתאים את ההגנות שלהן מבלי לכפות על כל הרשת מדיניות אחת.

## מה זה משנה

בפלטפורמה מרכזית, מדיניות אנטי-ספאם הופכת בדרך כלל למדיניות מוצר. כשזה קורה, חברה אחת מחליטה מה נחשב כחשבון חוקי, כפוסט חוקי או טווח הגעה מקובל.

Bitsocial שובר את הצימוד הזה:

- הפרוטוקול נשאר ניטרלי
- הקהילה בוחרת את האתגר
- אפליקציות מחליטות כיצד להציג או להפשט את האתגר הזה

זה מתאים יותר לרשת שנועדה לארח סוגים רבים של מוצרים חברתיים ולא ברירת מחדל עולמית אחת.
