# Флагманское приложение Bitsocial

Флагманское приложение Bitsocial — это предлагаемый первый клиент сети на основе профилей. Самое простое сравнение: по умолчанию знакомо, как X/Twitter, но с открытым слоем платформы.

Он будет добавлять профили, подписки, ответы, каналы, сообщества, уведомления и публичные беседы в реальном времени, сохраняя при этом заменяемые базовые сервисы. 5chan доказывает, что анонимные сообщества могут работать. Seedit движется к постоянному обсуждению. Флагманское приложение позволит перенести эти сетевые эффекты в основную социальную ленту, не делая ни одну компанию владельцем графика.

На этой странице описывается направление продукта, а не спецификация закрытого выпуска. Точный интерфейс, канал по умолчанию, рекламная модель, функции искусственного интеллекта и торговая площадка RPC могут меняться по мере развития протокола и ранних приложений.

## Что это должно доказать

Приложение должно доказать, что социальная сеть на основе профилей может не стать кастодиальной платформой:

- пользователи могут владеть личностями и профилями
- сообщества и узлы профилей могут оставаться одноранговыми
- сообщества могут переносить сетевые эффекты на клиентов Bitsocial
- Поставщики RPC могут сделать приложение удобным, не принимая на себя ответственность за его хранение.
- алгоритмы подачи могут быть дополнительными услугами вместо закона платформы
- другие клиенты по-прежнему могут конкурировать за ту же сеть

Дело не в том, чтобы сделать единственный клиент Bitsocial. Цель состоит в том, чтобы создать первый широкий клиент, который покажет, насколько далеко может простираться протокол.

## Знакомый по умолчанию, заменяемый по дизайну

Опыт по умолчанию должен конкурировать с основными приложениями для публичных бесед: быстрая домашняя лента, подписки, ответы, распространение в стиле репостов, сообщества, уведомления, поиск и ранжированный просмотр «Для вас», который работает немедленно.

Bitsocial Forge может запустить первый RPC по умолчанию и службу подачи. По умолчанию это может включать ранжированный канал и рекламу, чтобы приложение выглядело завершенным с первого же дня, вместо того, чтобы просить обычных пользователей собрать весь стек самостоятельно.

Разница в том, что дефолт не должен стать тюрьмой. Пользователь должен иметь возможность переключать RPC, каналы, экземпляры, системы ранжирования, рекламу и поставщиков обнаружения или полностью удалять ранжирование. Приложение может быть самоуверенным при первом запуске, сохраняя при этом возможность замены всех основных сервисов.

Это делает приложение более настраиваемым, чем обычную платформу. Один пользователь может сохранить рейтинговый канал по умолчанию с рекламой. Другой может использовать хронологический канал без рейтинга. Другой может выбрать RPC, ориентированный на конфиденциальность, общественную службу поиска, платный канал без рекламы или нишевый алгоритм, созданный для конкретной субкультуры.

## Межклиентские сообщества

Сообщества должны быть гораздо важнее, чем изолированные группы внутри одного приложения.

В X/Twitter сообщества ограничены внутри X. Они могут быть полезны, но наследуют ограничения одной платформы, одной системы учетных записей, одного набора рекомендаций и одной поверхности продукта.

Сообщество Bitsocial можно создавать, размещать, обнаруживать и использовать с помощью разных клиентов. Это означает, что флагманское приложение может отображать сообщества и публикации из более широкой сети Bitsocial, а не только от пользователей, которые начали работу во флагманском приложении. Сообщество может одновременно иметь активность со стороны клиента имиджборда, дискуссионного клиента в стиле Reddit, нишевого клиента форума, мобильного приложения и флагманского приложения.

В этом заключается основное преимущество сетевого эффекта: один клиент может показаться знакомым обычным пользователям, сохраняя при этом ценность от многих клиентов, узлов сообщества, поставщиков RPC и независимых сервисов.

## Дополнительные алгоритмы подачи

Флагманское приложение не должно навязывать всем одну глобальную систему рейтинга.

Алгоритмы подачи должны быть добровольными. Пользователь может выбрать алгоритм на торговой площадке, сменить поставщика, использовать алгоритм компании, использовать алгоритм анонимного оператора, использовать алгоритм, созданный сообществом, запустить личный алгоритм или вообще не использовать алгоритм.

Публичные поставщики RPC являются естественной площадкой для конкуренции этих услуг. Они могут индексировать, ранжировать и рекомендовать контент, но не должны владеть пользователем или профилем.

Эти сервисы также могут конкурировать по форме самого приложения. Один RPC может предоставлять ранжированный канал с рекламой. Другой может предоставить неранжированную хронологическую ленту. Другой может специализироваться на конфиденциальности, переводе, модерации, открытии сообщества или нишевом социальном графе.

Если экономика сработает, службы каналов, поддерживаемые RPC, могут добавить функции искусственного интеллекта, аналогичные тем, которые основные платформы пытаются внедрить в свои каналы: автоматические переводы, резюме, ответы с помощью ботов, ответы на поиск, помощь в модерации или контекст в стиле комментариев сообщества.

Эти функции должны быть выбором сервиса, а не требованиями протокола. RPC по умолчанию может конкурировать, предлагая более богатый канал, но пользователи и конкурирующие клиенты по-прежнему должны иметь возможность выбирать более простые, частные, хронологические, без рекламы или специфичные для сообщества альтернативы.

## RPC, не связанный с лишением свободы

Каждый пользователь должен иметь возможность участвовать в качестве полноценного однорангового узла через RPC, не передавая поставщику RPC право собственности на свою личность или профиль.

Размещенный путь имеет значение, поскольку большинство пользователей не начинают с запуска сервера. Путь выхода имеет не меньшее значение: пользователь должен иметь возможность перейти к своему собственному узлу профиля на оборудовании с низкими характеристиками, включая Raspberry Pi, когда захочет.

В этом разница между удобством и опекой.

## Почему оно может стать универсальным приложением

Если Bitsocial Chain предоставит приложениям надежные имена, платежи, чаевые, награды и другие финансовые возможности, флагманское приложение может стать чем-то большим, чем просто клиентом каналов.

Важным ограничением является то, что приложение не должно стать новым владельцем сети. Это может быть крупный клиент, возможно, даже самый популярный клиент, но при этом остается место для конкурирующих приложений, конкурирующих RPC, конкурирующих алгоритмов подачи и узлов автономных профилей.
