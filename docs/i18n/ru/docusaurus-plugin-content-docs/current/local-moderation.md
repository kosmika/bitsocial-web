# Локальная модерация, а не глобальные баны

Bitsocial не свободен от модерации. Это модерация без суперадминистратора уровня протокола.

Владельцы сообществ по-прежнему устанавливают правила для своих пространств. Приложения по-прежнему решают, что им индексировать, ранжировать или выделять. Чего Bitsocial не включает в себя, так это глобального органа власти, который может конфисковать сообщество или стереть личность из самой сети.

Более подробное описание прав собственности сообщества, публикаций, задач и подписей модераторов на простом английском языке можно найти в статье [Полное объяснение протокола Bitsocial для непрофессионалов.](./layman-protocol-explanation.md).

## Как работает модерация

- Владелец сообщества может модерировать это сообщество.
- Приложение может выбирать, что индексировать или показывать.
- Пользователь может перейти в другое сообщество или другое приложение.

Если одно приложение фильтрует вас, другое приложение все равно может вас показать. Если одно сообщество отвергает вас, другое сообщество может принять вас или вы можете создать свое собственное.

## Чего Bitsocial избегает

Bitsocial избегает знакомого узкого места платформы, когда одна компания контролирует:

- социальный граф
- база данных модерации
- поверхность открытия
- пространство имен учетной записи

Именно этот центральный момент превращает решение о политике в отношении продукта в общесетевой запрет.

## Практический компромисс

Локальная модерация не означает, что все приложения будут выглядеть одинаково. Это означает, что последствия модерации остаются ограниченными:

- правила сообщества применяются внутри этого сообщества
- правила приложения применяются внутри этого приложения
- ни один из них автоматически не становится протокольным законом

Именно это и пытается сохранить принцип «отсутствия глобальных запретов».
