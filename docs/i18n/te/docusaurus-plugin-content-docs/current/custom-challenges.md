# కస్టమ్ యాంటీ-స్పామ్ సవాళ్లు

Bitsocial ఒక సార్వత్రిక స్పామ్ రక్షణను ఊహించదు. ప్రతి సంఘం చెల్లుబాటు అయ్యే పోస్ట్ లేదా ప్రత్యుత్తరానికి ఏమి అవసరమో నిర్ణయించుకోవచ్చు మరియు ఆ సవాలు ఆపరేటర్ కోరుకున్నంత తేలికగా లేదా అభిప్రాయాన్ని కలిగి ఉండవచ్చు.

కమ్యూనిటీ చిరునామాల నుండి సవాలు-గేటెడ్ పోస్టింగ్ వరకు పూర్తి సాదా-ఇంగ్లీష్ ఫ్లో కోసం, చదవండి [బిట్సోషల్ ప్రోటోకాల్ యొక్క పూర్తి సామాన్య వివరణ](./layman-protocol-explanation.md).

## ఎంత సవాలుగా ఉండవచ్చు

- ఒక క్యాప్చా
- ఖాతా-వయస్సు లేదా కీర్తి తనిఖీ
- SMS ధృవీకరణ
- చిన్న చెల్లింపు
- టోకెన్ లేదా NFT అవసరం
- IP ఆధారిత నియమాలు
- అనుమతించబడిన జాబితా
- కోడ్‌లో వ్యక్తీకరించబడే ఏదైనా ఇతర విధానం

## ఇది ఎందుకు ముఖ్యం

ప్రోటోకాల్-వైడ్ మోడరేషన్ అథారిటీకి అవుట్‌సోర్స్ చేయడానికి బదులుగా స్పామ్ రెసిస్టెన్స్ కమ్యూనిటీకి స్థానికంగా ఉంటుంది. కమ్యూనిటీని హోస్ట్ చేసే నోడ్ పీర్-టు-పీర్ ఛాలెంజ్‌ను కమ్యూనికేట్ చేయగలదు, అంటే కమ్యూనిటీలు మొత్తం నెట్‌వర్క్‌ను ఒకే విధానంలోకి నెట్టకుండా తమ రక్షణను స్వీకరించగలవు.

## ఇది ఏమి మారుస్తుంది

కేంద్రీకృత ప్లాట్‌ఫారమ్‌లో, యాంటీ-స్పామ్ విధానం సాధారణంగా ఉత్పత్తి విధానం అవుతుంది. అది జరిగినప్పుడు, చెల్లుబాటు అయ్యే ఖాతా, చెల్లుబాటు అయ్యే పోస్ట్ లేదా ఆమోదయోగ్యమైన రీచ్‌గా ఏది లెక్కించబడుతుందో ఒక కంపెనీ నిర్ణయిస్తుంది.

బిట్సోషల్ ఆ కలయికను విచ్ఛిన్నం చేస్తుంది:

- ప్రోటోకాల్ తటస్థంగా ఉంటుంది
- సమాజం సవాలును ఎంచుకుంటుంది
- యాప్‌లు ఆ ఛాలెంజ్‌ని ఎలా ప్రదర్శించాలో లేదా వియుక్తంగా ఎలా చేయాలో నిర్ణయిస్తాయి

ఒక గ్లోబల్ డిఫాల్ట్ కాకుండా అనేక రకాల సామాజిక ఉత్పత్తులను హోస్ట్ చేయడానికి ఉద్దేశించిన నెట్‌వర్క్‌కు ఇది బాగా సరిపోతుంది.
