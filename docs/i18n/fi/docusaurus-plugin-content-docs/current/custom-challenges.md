# Mukautetut roskapostin torjuntahaasteet

Bitsocial ei oleta yhtä yleistä roskapostisuojausta. Jokainen yhteisö voi päättää, mitä pätevä viesti tai vastaus vaatii, ja haaste voi olla niin kevyt tai mielikuvituksellinen kuin operaattori haluaa.

Lue koko pelkkä englanninkielinen kulku yhteisön osoitteista haasteisiin liittyvään postaukseen [Täydellinen maallikon selitys Bitsocial-protokollasta](./layman-protocol-explanation.md).

## Mikä haaste voi olla

- Captcha
- Tilin iän tai maineen tarkistus
- SMS-vahvistus
- Pieni maksu
- Token- tai NFT-vaatimus
- IP-pohjaiset säännöt
- Sallittujen luettelo
- Mikä tahansa muu käytäntö, joka voidaan ilmaista koodilla

## Miksi tällä on väliä

Roskapostin vastustuskyky pysyy paikallisena yhteisössä sen sijaan, että se olisi ulkoistettu protokollan laajuiselle valvontaviranomaiselle. Yhteisöä isännöivä solmu voi viestiä haasteesta vertaisverkkoon, mikä tarkoittaa, että yhteisöt voivat mukauttaa puolustustaan ​​pakottamatta koko verkkoa yhteen käytäntöön.

## Mitä tämä muuttaa

Keskitetyllä alustalla roskapostin vastaisesta käytännöstä tulee yleensä tuotepolitiikka. Kun näin tapahtuu, yksi yritys päättää, mikä lasketaan kelvolliseksi tiliksi, kelvolliseksi postaukseksi tai hyväksyttäväksi tavoittavuudeksi.

Bitsocial katkaisee tämän kytkennän:

- protokolla pysyy neutraalina
- yhteisö valitsee haasteen
- sovellukset päättävät, kuinka haaste esitetään tai abstraktoidaan

Tämä sopii paremmin verkostoon, joka on tarkoitettu isännöimään monenlaisia sosiaalisia tuotteita, kuin yhteen maailmanlaajuiseen oletusarvoon.
