# Desafíos antispam personalizados

Bitsocial no asume una defensa universal contra el spam. Cada comunidad puede decidir qué requiere una publicación o respuesta válida, y ese desafío puede ser tan ligero u obstinado como quiera el operador.

Para conocer el flujo completo en inglés sencillo, desde las direcciones de la comunidad hasta las publicaciones con acceso a desafíos, lea [Una explicación completa y sencilla del protocolo Bitsocial.](./layman-protocol-explanation.md).

## ¡Qué desafío puede ser!

- Un captcha
- Una verificación de antigüedad o reputación de la cuenta
- Verificación por SMS
- un pequeño pago
- Un requisito de token o NFT
- reglas basadas en IP
- Una lista de permitidos
- Cualquier otra política que pueda expresarse en código.

## Por qué esto importa

La resistencia al spam permanece local en la comunidad en lugar de subcontratarse a una autoridad de moderación de todo el protocolo. El nodo que alberga la comunidad puede comunicar el desafío entre pares, lo que significa que las comunidades pueden adaptar sus defensas sin obligar a toda la red a adoptar una sola política.

## ¿Qué cambia esto?

En una plataforma centralizada, la política antispam suele convertirse en política de producto. Cuando eso sucede, una empresa decide qué se considera una cuenta válida, una publicación válida o un alcance aceptable.

Bitsocial rompe ese acoplamiento:

- el protocolo se mantiene neutral
- la comunidad elige el desafío
- las aplicaciones deciden cómo presentar o abstraer ese desafío

Esta es una mejor opción para una red destinada a albergar muchos tipos de productos sociales en lugar de un valor predeterminado global.
