# Benutzerdefinierte Anti-Spam-Herausforderungen

Bitsocial geht nicht von einer universellen Spam-Abwehr aus. Jede Community kann entscheiden, was ein gültiger Beitrag oder eine gültige Antwort erfordert, und diese Herausforderung kann so leicht oder rechthaberisch sein, wie der Betreiber es wünscht.

Den vollständigen Ablauf in Klartext von Community-Adressen bis hin zu herausforderungsgesteuerten Beiträgen finden Sie hier [Eine vollständige Erklärung des Bitsocial-Protokolls für Laien](./layman-protocol-explanation.md).

## Was für eine Herausforderung kann sein

- Ein Captcha
- Eine Überprüfung des Kontoalters oder der Reputation
- SMS-Verifizierung
- Eine kleine Zahlung
- Eine Token- oder NFT-Anforderung
- IP-basierte Regeln
- Eine Zulassungsliste
- Jede andere Richtlinie, die im Code ausgedrückt werden kann

## Warum das wichtig ist

Die Spam-Resistenz bleibt lokal in der Community und wird nicht an eine protokollweite Moderationsbehörde ausgelagert. Der Knoten, der die Community hostet, kann die Herausforderung Peer-to-Peer kommunizieren, was bedeutet, dass Communities ihre Abwehrmaßnahmen anpassen können, ohne das gesamte Netzwerk einer Richtlinie zu unterwerfen.

## Was sich dadurch ändert

Auf einer zentralisierten Plattform wird die Anti-Spam-Richtlinie normalerweise zur Produktrichtlinie. In diesem Fall entscheidet ein Unternehmen, was als gültiger Account, gültiger Beitrag oder akzeptable Reichweite gilt.

Bitsocial bricht diese Kopplung:

- Das Protokoll bleibt neutral
- Die Community wählt die Herausforderung
- Apps entscheiden, wie sie diese Herausforderung darstellen oder abstrahieren

Dies passt besser zu einem Netzwerk, das viele Arten sozialer Produkte hosten soll, als zu einem globalen Standard.
