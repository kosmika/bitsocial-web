# Lokal na Moderation, Hindi Global Bans

Ang bitsocial ay hindi moderation-free. Ito ay moderation na walang protocol-level super-admin.

Nagtatakda pa rin ang mga may-ari ng komunidad ng mga panuntunan para sa kanilang sariling mga espasyo. Ang mga app ay nagpapasya pa rin kung ano ang kanilang ini-index, niraranggo, o na-highlight. Ang hindi kasama sa Bitsocial ay isang pandaigdigang awtoridad na maaaring kumpiskahin ang isang komunidad o magbura ng isang pagkakakilanlan mula sa mismong network.

Para sa isang mas malawak na payak na Ingles na walkthrough ng pagmamay-ari ng komunidad, pag-publish, mga hamon, at mga lagda ng moderator, basahin [Isang kumpletong paliwanag ng karaniwang tao sa Bitsocial protocol](./layman-protocol-explanation.md).

## Paano gumagana ang moderation

- Maaaring i-moderate ng isang may-ari ng komunidad ang komunidad na iyon
- Maaaring piliin ng isang app kung ano ang ini-index o ipinapakita nito
- Maaaring lumipat ang isang user sa ibang komunidad o ibang app

Kung sinasala ka ng isang app, maaari pa ring piliin ng isa pang app na ipakita sa iyo. Kung tatanggihan ka ng isang komunidad, maaaring tanggapin ka ng ibang komunidad, o maaari kang lumikha ng sarili mong komunidad.

## Ang iniiwasan ng Bitsocial

Iniiwasan ng Bitsocial ang pamilyar na platform choke point kung saan kinokontrol ng isang kumpanya ang:

- ang social graph
- ang database ng moderation
- ibabaw ng pagtuklas
- ang namespace ng account

Ang pangunahing puntong iyon ay kung bakit ang isang desisyon sa patakaran sa produkto ay isang pagbabawal sa buong network.

## Ang praktikal na tradeoff

Ang lokal na pag-moderate ay hindi nangangahulugan na ang bawat app ay magmumukhang magkapareho. Nangangahulugan ito na ang mga kahihinatnan ng pag-moderate ay nananatiling saklaw:

- Nalalapat ang mga alituntunin ng komunidad sa loob ng komunidad na iyon
- Nalalapat ang mga panuntunan sa app sa loob ng app na iyon
- ni awtomatikong nagiging protocol law

Iyan ang sinusubukang pangalagaan ng "walang pandaigdigang pagbabawal".
