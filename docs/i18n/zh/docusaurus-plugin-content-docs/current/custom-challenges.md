# 自定义反垃圾邮件挑战

Bitsocial 并不假设存在一种通用的垃圾邮件防御机制。每个社区都可以决定有效的帖子或回复需要什么，并且该挑战可以是轻量级的，也可以是固执己见的，如操作员想要的那样。

有关从社区地址到挑战门控发帖的完整简单英语流程，请阅读 [Bitsocial 协议的完整外行解释](./layman-protocol-explanation.md).

## 这是多么大的挑战

- 验证码
- 帐户年龄或声誉检查
- 短信验证
- 少量付款
- 代币或 NFT 要求
- 基于IP的规则
- 允许名单
- 可以用代码表达的任何其他策略

## 为什么这很重要

垃圾邮件抵抗保留在社区本地，而不是外包给协议范围内的审核机构。托管社区的节点可以点对点地传达挑战，这意味着社区可以调整其防御措施，而无需强迫整个网络采用一项策略。

## 这改变了什么

在中心化平台上，反垃圾邮件策略通常成为产品策略。当这种情况发生时，一家公司会决定什么算作有效帐户、有效帖子或可接受的覆盖范围。

Bitsocial 打破了这种耦合：

- 协议保持中立
- 社区选择挑战
- 应用程序决定如何呈现或抽象该挑战

这更适合用于托管多种社交产品的网络，而不是一种全球默认产品。
