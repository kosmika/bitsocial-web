# 本地监管，而非全球禁令

Bitsocial 并不是没有节制的。这是没有协议级超级管理员的调节。

社区业主仍然为自己的空间制定规则。应用程序仍然决定它们索引、排名或突出显示的内容。 Bitsocial 不包括一个可以没收社区或从网络本身删除身份的全球权威机构。

有关社区所有权、发布、挑战和主持人签名的更广泛的简单英语演练，请阅读 [Bitsocial 协议的完整外行解释](./layman-protocol-explanation.md).

## 节制如何发挥作用

- 社区所有者可以管理该社区
- 应用程序可以选择索引或显示的内容
- 用户可以移动到另一个社区或另一个应用程序

如果一个应用程序过滤了您，另一应用程序仍然可以选择向您显示。如果一个社区拒绝你，另一个社区可以接受你，或者你可以创建自己的社区。

## Bitsocial 避免什么

Bitsocial 避免了常见的由一家公司控制的平台瓶颈：

- 社交图谱
- 审核数据库
- 发现面
- 帐户命名空间

这个中心点是将产品政策决策转变为全网络禁令的原因。

## 实际的权衡

本地审核并不意味着每个应用程序看起来都一样。这意味着适度的后果仍然是有限的：

- 社区规则适用于该社区内部
- 应用程序规则适用于该应用程序内部
- 两者都不会自动成为协议法

这就是“无全球禁令”试图维护的目标。
