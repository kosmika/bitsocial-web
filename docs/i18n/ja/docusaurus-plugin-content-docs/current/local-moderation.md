# グローバルな禁止ではなくローカルなモデレーション

Bitsocial はモデレーションフリーではありません。これは、プロトコル レベルのスーパー管理者なしでのモデレーションです。

コミュニティの所有者は依然として自分のスペースのルールを設定します。何をインデックス付けするか、ランク付けするか、ハイライトするかはアプリが決定します。 Bitsocial には、コミュニティを没収したり、ネットワーク自体から ID を消去したりできるグローバルな権限は含まれていません。

コミュニティの所有権、公開、チャレンジ、モデレーターの署名に関する広範な平易な英語のウォークスルーについては、以下を参照してください。 [Bitsocial プロトコルの完全な素人説明](./layman-protocol-explanation.md).

## モデレーションの仕組み

- コミュニティの所有者はそのコミュニティをモデレートできます
- アプリは、インデックスに登録する内容や表示する内容を選択できます
- ユーザーは別のコミュニティまたは別のアプリに移動できます

あるアプリがあなたをフィルタリングしても、別のアプリはあなたを表示することを選択できます。 1 つのコミュニティがあなたを拒否しても、別のコミュニティがあなたを受け入れることもできますし、独自のコミュニティを作成することもできます。

## Bitsocial が回避するもの

Bitsocial は、1 つの企業が管理するおなじみのプラットフォームの難所を回避します。

- ソーシャル グラフ
- モデレーション データベース
- ディスカバリ サーフェス
- アカウント ネームスペース

その中心点はこれにより、製品ポリシーの決定がネットワーク全体の禁止に変わります。

## 実際的なトレードオフ

ローカルのモデレーションは、すべてのアプリが同一に見えることを意味するわけではありません。これは、モデレーションの結果が限定されたままであることを意味します。

- コミュニティ ルールはそのコミュニティ内に適用されます
- アプリ ルールはそのアプリ内に適用されます
- 自動的にプロトコル法になることはありません

これが「世界的な禁止なし」が維持しようとしているものです。
