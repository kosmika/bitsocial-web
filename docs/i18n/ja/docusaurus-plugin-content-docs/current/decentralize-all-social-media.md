# すべてのソーシャル メディアを分散化

これは、Bitsocial マスタープランで説明される最終状態です。つまり、1 つのアプリではなく、ソーシャル メディアのあらゆる主要カテゴリに匹敵する Bitsocial アプリのエコシステムです。

## モデル

提案されているネットワークの成熟した状態では:

- 複数のパブリック RPC プロバイダーがプロファイル ノードをホストするために競合します
- ユーザーは引き続きセルフホストできます
- フィード アルゴリズムは、代わりにオプションのサービスになります必須のプラットフォーム法
- 収益化は、プラットフォームを存続させようとする銀行に依存しない
- アプリは製品の品質、発見、インターフェースのデザインで競争する

## 成功とはどのようなものか

ビットソーシャルアプリは、単一のニッチ分野だけでなく、さまざまなフォーマットでプラットフォームに挑戦できるべきである

- テキストとディスカッション
- イメージボード
- フォーラム
- クリエイター コミュニティ
- 短編ビデオおよび広範なメディア製品

目標は、1 つのユニバーサル インターフェイスをすべての人に強制することではありません。これは、多くのアプリが同じ基礎となるコミュニティ、アイデンティティ、ネットワークにアクセスできるようにするためです。

## ここで RPC レイヤーが重要な理由

パブリック RPC マーケットが存在すると、常時接続のコミュニティとプロファイルがより実用的になります。この層がなければ、セルフホスティングは多くのユーザーにとって、特にモバイルでは負担が大きすぎるままです。

## ここでネットワーク層が重要な理由

コンテンツの収益化が通常の銀行取引やプラットフォームの圧力によって遮断される可能性がある場合、エコシステムは脆弱なままです。 Bitsocial Chain は、長期的な競争の信頼性を高める経済層として提案されています。

## 主要な賭け

大きな賭けは、ソーシャル メディアを置き換え可能なものに分割できることです。レイヤー:

- プロトコル
- コミュニティ
- アイデンティティ
- ホスティング
- ディスカバリー
- 収益化
- アプリ

これらのレイヤーが置き換え可能になると、製品が機能するために単一の企業がソーシャル グラフ全体を所有する必要はなくなります。
