# Bitsocial Chain

Bitsocial Chain は、Bitsocial アプリの下に位置する共有アプリチェーン レイヤーのマスター プランで使用される名前です。

このアイデアは、ピアツーピアのソーシャル層を置き換えることではありません。これは、ソーシャル アプリが既存のプラットフォームと大規模に競合する場合に必要となる、共有の経済性と調整の基本要素を提供することです。

## 力を与えるということの意味

- `.bso` などの分散型 Bitsocial ドメイン
- アプリ間で流動性を共有する
- 耐久性のある収益化レール
- 銀行やプラットフォームにとってより困難な金融構造
- スタック全体を所有する 1 つの会社に依存しないネットワーク効果

## なぜそれが重要なのか

コミュニティとアイデンティティを分散化することは必要ですが、すべてのソーシャルメディアを分散化するだけでは十分ではありません。

経済層がまだ少数の集中化されたレールに依存している場合、アプリは引き続き圧力を受けたり、プラットフォームから外されたり、財政的に困窮したりしやすくなります。 Bitsocial Chain は、依存関係の第 2 層に対する提案された答えです。

## Bitsocial の残りの部分との関係

Bitsocial Chain は、ユーザーが毎日直接操作する製品としてではなく、Bitsocial アプリの共有インフラストラクチャとして提案されています。

一般向けの結果は次のようになります。

- コミュニティはピアツーピアを維持します
- アプリは差別化を維持する
- 収益化とネーミングがより耐久性のあるものになる
- 一元化されたプラットフォーム所有者を再作成することなく、エコシステム全体で価値を移動できる

## これがフェーズ 4 にある理由

マスタープランでは、これを以前のアプリレベルの証明ポイントの後に配置します。

1. イメージボードを分散化する
2. Bitsocial RPC を起動する
3. フォーラムを分散化する
4. Bitsocial Chainを立ち上げる

実際のアプリと実際のユーザーの行動がネットワーク上にすでに存在すると、エコシステム レベルの調整レイヤーがより有用になるため、この順序は理にかなっています。

## 設計原則

Bitsocial Chain はまだ立ち上がった製品ではなく、提案段階のインフラであるため、現在の計画は次の点で節度を保つべきです。

- まずはアプリとコミュニティです。ネットワーク層は Bitsocial アプリ内の実際の利用に従うべきであり、投機的な金融機能が先行すべきではありません。
- 誇大宣伝より実際の貢献を重視します。将来 Bitsocial がコミュニティ、クリエイター、またはクライアント単位の資産を追加するなら、それらは模倣的なローンチや短期売買ではなく、参加と長期的な構築を報いるべきです。
- キュレーションは認められます。Bitsocial アプリは、ランキング、既定値、発見性、展開のタイミングを調整し、カジノ的な振る舞いよりも持続的なコミュニティを優先できます。
- 段階的な展開は健全です。エコシステムに十分なユーザー、モデレーション手段、製品としての明確さがそろうまでは、appchain ネイティブな金融機能を遅らせるのは妥当です。
- 正確な仕組みはまだ未確定です。このページは Bitsocial Chain の役割を説明するものであり、トークンランチャー、手数料式、最終的なトークノミクスについての確定した約束ではありません。
