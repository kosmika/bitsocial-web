# กระจายอำนาจโซเชียลมีเดียทั้งหมด

นี่คือสถานะสุดท้ายที่อธิบายไว้ในแผนแม่บทของ Bitsocial ไม่ใช่แอปเดียว แต่เป็นระบบนิเวศของแอป Bitsocial ที่สามารถแข่งขันกับโซเชียลมีเดียหลักทุกประเภทได้

## โมเดล

ในสถานะที่สมบูรณ์ของเครือข่ายที่เสนอ:

- ผู้ให้บริการ RPC สาธารณะหลายรายแข่งขันกันเพื่อโฮสต์โปรไฟล์โหนด
- ผู้ใช้ยังสามารถโฮสต์ด้วยตนเองได้
- อัลกอริธึมฟีดกลายเป็นบริการเสริมแทนกฎหมายแพลตฟอร์มบังคับ
- การสร้างรายได้ไม่ได้ขึ้นอยู่กับธนาคารที่เต็มใจที่จะรักษาแพลตฟอร์มให้คงอยู่ต่อไป
- แอพแข่งขันกันในด้านคุณภาพผลิตภัณฑ์ การค้นพบ และการออกแบบอินเทอร์เฟซ

## ความสำเร็จจะเป็นอย่างไร.

แอป Bitsocial ควรสามารถท้าทายแพลตฟอร์มข้ามรูปแบบได้ ไม่ใช่แค่ในช่องเดียว:

- ข้อความและการอภิปราย
- บอร์ดรูปภาพ
- ฟอรั่ม
- ชุมชนผู้สร้าง
- วิดีโอแบบสั้นและผลิตภัณฑ์สื่อในวงกว้าง

เป้าหมายไม่ใช่การบังคับใช้อินเทอร์เฟซสากลเดียวสำหรับทุกคน เป็นการให้แอปจำนวนมากเข้าถึงชุมชน ข้อมูลประจำตัว และเครือข่ายเดียวกัน

## เหตุใดเลเยอร์ RPC จึงมีความสำคัญที่นี่

ชุมชนและโปรไฟล์ที่เปิดตลอดเวลาจะใช้งานได้จริงมากขึ้นเมื่อมีตลาด RPC สาธารณะอยู่ หากไม่มีเลเยอร์นั้น การโฮสต์ด้วยตนเองยังคงมีความขัดแย้งสูงเกินไปสำหรับผู้ใช้จำนวนมาก โดยเฉพาะบนมือถือ

## เหตุใดเลเยอร์เครือข่ายจึงมีความสำคัญที่นี่

หากการสร้างรายได้จากเนื้อหาสามารถถูกตัดออกโดยธนาคารทั่วไปหรือแรงกดดันจากแพลตฟอร์ม ระบบนิเวศก็ยังคงเปราะบาง Bitsocial Chain ถูกเสนอให้เป็นชั้นทางเศรษฐกิจที่ทำให้การแข่งขันระยะยาวมีความน่าเชื่อถือมากขึ้น

## เดิมพันหลัก

เดิมพันใหญ่คือโซเชียลมีเดียสามารถแบ่งออกเป็นเลเยอร์ที่ทดแทนได้:

- โปรโตคอล
- ชุมชน
- ตัวตน
- โฮสติ้ง
- การค้นพบ
- การสร้างรายได้
- แอพ

เมื่อเลเยอร์เหล่านั้นสามารถเปลี่ยนได้ ไม่มีบริษัทใดที่จะต้องเป็นเจ้าของกราฟโซเชียลทั้งหมดเพื่อให้ผลิตภัณฑ์ทำงานได้
