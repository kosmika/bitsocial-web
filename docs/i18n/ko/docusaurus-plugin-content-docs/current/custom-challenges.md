# 맞춤형 스팸 방지 과제

Bitsocial은 하나의 보편적인 스팸 방어를 가정하지 않습니다. 각 커뮤니티는 유효한 게시물이나 답글에 필요한 것이 무엇인지 결정할 수 있으며 해당 챌린지는 운영자가 원하는 만큼 간단하거나 독선적일 수 있습니다.

커뮤니티 주소부터 챌린지 제한 게시까지 전체 일반 영어 흐름을 보려면 다음을 읽어보세요. [Bitsocial 프로토콜에 대한 완전한 평신도 설명](./layman-protocol-explanation.md).

## 챌린지는

- 보안 문자
- 계정 연령 또는 평판 확인
- SMS 확인
- 소액 결제
- 토큰 또는 NFT 요구 사항
- IP 기반 규칙
- 허용 목록
- 코드로 표현할 수 있는 기타 정책

## 이것이 중요한 이유

스팸 저항은 프로토콜 전체 중재 기관에 아웃소싱되는 대신 커뮤니티에 로컬로 유지됩니다. 커뮤니티를 호스팅하는 노드는 챌린지를 피어 투 피어로 전달할 수 있습니다. 이는 커뮤니티가 전체 네트워크를 하나의 정책으로 강제하지 않고도 방어를 조정할 수 있음을 의미합니다.

## 이 변경 사항

중앙 집중식 플랫폼에서 스팸 방지 정책은 일반적으로 제품 정책이 됩니다. 그런 일이 발생하면 한 회사가 유효한 계정, 유효한 게시물 또는 허용 가능한 도달 범위를 결정합니다.

Bitsocial은 이러한 결합을 깨뜨립니다.

- 프로토콜은 중립을 유지합니다
- 커뮤니티는 도전 과제를 선택합니다
- 앱은 해당 도전 과제를 제시하거나 추상화하는 방법을 결정합니다

이는 하나의 글로벌 기본 설정보다는 다양한 종류의 소셜 제품을 호스팅하기 위한 네트워크에 더 적합합니다.
