# 전역 금지가 아닌 지역 조정

Bitsocial은 중재가 필요하지 않습니다. 프로토콜 수준의 최고 관리자가 없는 중재입니다.

커뮤니티 소유자는 여전히 자신의 공간에 대한 규칙을 설정합니다. 앱은 여전히 ​​색인, 순위, 강조 표시 항목을 결정합니다. Bitsocial에는 커뮤니티를 압수하거나 네트워크 자체에서 신원을 삭제할 수 있는 글로벌 권한이 포함되어 있지 않습니다.

커뮤니티 소유권, 게시, 챌린지 및 중재자 서명에 대한 더 광범위한 일반 영어 연습을 보려면 다음을 읽어보세요. [Bitsocial 프로토콜에 대한 완전한 평신도 설명](./layman-protocol-explanation.md).

## 조정 작동 방식

- 커뮤니티 소유자는 해당 커뮤니티를 조정할 수 있습니다
- 앱은 인덱싱하거나 표시할 내용을 선택할 수 있습니다
- 사용자는 다른 커뮤니티 또는 다른 앱으로 이동할 수 있습니다

한 앱이 귀하를 필터링하는 경우에도 다른 앱이 귀하를 표시하도록 선택할 수 있습니다. 한 커뮤니티가 귀하를 거부하면 다른 커뮤니티가 귀하를 받아들이거나 귀하가 자신만의 커뮤니티를 만들 수 있습니다.

## Bitsocial이 피하는 것

Bitsocial은 한 회사가 통제하는 친숙한 플랫폼 관문을 피합니다:

- 소셜 그래프
- 조정 데이터베이스
- 검색 표면
- 계정 네임스페이스

이 중심점이 바로 전환입니다. 네트워크 전체 금지에 대한 제품 정책 결정.

## 실질적인 절충

로컬 조정이 모든 앱이 동일하게 보인다는 의미는 아닙니다. 이는 중재의 결과 범위가 일정하게 유지됨을 의미합니다.

- 커뮤니티 규칙은 해당 커뮤니티 내에 적용됩니다
- 앱 규칙은 해당 앱 내에 적용됩니다
- 자동으로 프로토콜 법이 되지도 않습니다

이것이 "전역 금지 없음"이 보존하려고 하는 것입니다.
