# Локальна модерація, а не глобальні заборони

Bitsocial не вимагає модерації. Це модерація без суперадміністратора на рівні протоколу.

Власники спільнот все ще встановлюють правила для своїх власних просторів. Додатки все ще вирішують, що їм індексувати, ранжувати чи виділяти. Чого Bitsocial не включає, так це глобальної влади, яка може конфіскувати спільноту або стерти особистість із самої мережі.

Щоб отримати ширший опис володіння спільнотою, публікації, викликів і підписів модераторів простою англійською мовою, прочитайте [Повне непрофесіональне пояснення протоколу Bitsocial](./layman-protocol-explanation.md).

## Як працює модерація

- Власник спільноти може модерувати цю спільноту
- Програма може вибирати, що вона індексує або показує
- Користувач може перейти до іншої спільноти або іншої програми

Якщо одна програма фільтрує вас, інша програма все одно може показувати вас. Якщо одна спільнота відхиляє вас, інша спільнота може прийняти вас, або ви можете створити власну.

## Чого уникає Bitsocial

Bitsocial уникає знайомої проблеми платформи, коли одна компанія контролює:

- соціальний граф
- база даних модерації
- поверхню відкриття
- простір імен облікового запису

Саме ця центральна точка перетворює рішення щодо політики продукту на заборону всієї мережі.

## Практичний компроміс

Локальна модерація не означає, що кожна програма виглядатиме однаково. Це означає, що наслідки модерації залишаються в силі:

- правила спільноти застосовуються всередині цієї спільноти
- правила програми застосовуються всередині цієї програми
- ні те, ні інше не стає автоматично протокольним законом

Саме це намагається зберегти «без глобальних заборон».
