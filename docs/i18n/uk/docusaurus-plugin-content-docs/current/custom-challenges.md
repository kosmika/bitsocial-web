# Спеціальні виклики захисту від спаму

Bitsocial не передбачає універсального захисту від спаму. Кожна спільнота може вирішити, що вимагає дійсний допис чи відповідь, і цей виклик може бути настільки легким чи впевненим, як того хоче оператор.

Прочитайте повний потік простою англійською мовою від адрес спільноти до опублікування через виклик [Повне непрофесіональне пояснення протоколу Bitsocial](./layman-protocol-explanation.md).

## Яким може бути виклик

- Captcha
- Перевірка віку або репутації облікового запису
- SMS перевірка
- Невелика оплата
- Вимога токена або NFT
- Правила на основі IP
- Білий список
- Будь-яка інша політика, яка може бути виражена в коді

## Чому це важливо

Стійкість до спаму залишається локальною для спільноти, замість того, щоб передаватись на аутсорсинг органу модерації, що діє на рівні протоколу. Вузол, на якому розміщується спільнота, може передавати виклик одноранговому зв’язку, що означає, що спільноти можуть адаптувати свої засоби захисту, не змушуючи всю мережу застосовувати одну політику.

## Що це змінює

На централізованій платформі політика боротьби зі спамом зазвичай стає політикою продукту. Коли це відбувається, одна компанія вирішує, що вважати дійсним обліковим записом, дійсною публікацією чи прийнятним охопленням.

Bitsocial розриває цей зв’язок:

- протокол залишається нейтральним
- громада вибирає виклик
- програми вирішують, як представити чи абстрагувати цей виклик

Це краще підходить для мережі, призначеної для розміщення багатьох видів соціальних продуктів, а не для однієї глобальної стандартної мережі.
