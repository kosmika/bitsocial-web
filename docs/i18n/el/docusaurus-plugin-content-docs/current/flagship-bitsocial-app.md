# Εμβληματική εφαρμογή Bitsocial

Η κορυφαία εφαρμογή Bitsocial είναι ο πρώτος προτεινόμενος πελάτης που βασίζεται σε προφίλ για το δίκτυο. Η απλούστερη σύγκριση είναι: εξοικειωμένοι όπως το X/Twitter από προεπιλογή, αλλά με το στρώμα της πλατφόρμας σπασμένο ανοιχτό.

Θα προσθέσει προφίλ, παρακολούθηση, απαντήσεις, ροές δεδομένων, κοινότητες, ειδοποιήσεις και δημόσια συνομιλία σε πραγματικό χρόνο, διατηρώντας παράλληλα τις υποκείμενες υπηρεσίες αντικαταστάσιμες. Το 5chan αποδεικνύει ότι οι ανώνυμες κοινότητες μπορούν να λειτουργήσουν. Το Sedit κινείται προς μια επίμονη συζήτηση. Η ναυαρχίδα της εφαρμογής θα έφερε αυτά τα εφέ δικτύου σε μια κύρια ροή κοινωνικής δικτύωσης χωρίς να καθιστά μια εταιρεία κάτοχο του γραφήματος.

Αυτή η σελίδα περιγράφει την κατεύθυνση του προϊόντος, όχι μια κλειδωμένη προδιαγραφή έκδοσης. Η ακριβής διεπαφή, η προεπιλεγμένη ροή, το μοντέλο διαφήμισης, οι λειτουργίες AI και η αγορά RPC μπορούν να αλλάξουν καθώς ωριμάζουν το πρωτόκολλο και οι πρώιμες εφαρμογές.

## Τι πρέπει να αποδείξει

Η εφαρμογή θα πρέπει να αποδεικνύει ότι ένα κοινωνικό δίκτυο που βασίζεται σε προφίλ μπορεί να αποφύγει να γίνει πλατφόρμα φύλαξης:

- Οι χρήστες μπορούν να κατέχουν ταυτότητες και προφίλ
- οι κοινότητες και οι κόμβοι προφίλ μπορούν να παραμείνουν peer-to-peer
- Οι κοινότητες μπορούν να μεταφέρουν εφέ δικτύου σε πελάτες Bitsocial
- Οι πάροχοι RPC μπορούν να κάνουν την εφαρμογή βολική χωρίς να αναλαμβάνουν την επιμέλεια
- Οι αλγόριθμοι τροφοδοσίας μπορούν να είναι προαιρετικές υπηρεσίες αντί του νόμου της πλατφόρμας
- άλλοι πελάτες μπορούν ακόμα να ανταγωνίζονται για το ίδιο δίκτυο

Το θέμα δεν είναι να κάνετε τον μοναδικό πελάτη Bitsocial. Το θέμα είναι να φτιάξουμε τον πρώτο ευρύ πελάτη που δείχνει πόσο μπορεί να εκτείνεται το πρωτόκολλο.

## Γνωστό από προεπιλογή, αντικαταστάσιμο από το σχεδιασμό

Η προεπιλεγμένη εμπειρία θα πρέπει να είναι ανταγωνιστική με τις κύριες εφαρμογές δημόσιας συνομιλίας: γρήγορη ροή στο σπίτι, follows, απαντήσεις, διανομή σε στυλ αναδημοσίευσης, κοινότητες, ειδοποιήσεις, αναζήτηση και μια κατάταξη προβολή για εσάς που λειτουργεί άμεσα.

Το Bitsocial Forge μπορεί να εκτελέσει την πρώτη προεπιλεγμένη υπηρεσία RPC και τροφοδοσίας. Αυτή η προεπιλογή μπορεί να περιλαμβάνει μια ταξινομημένη ροή και διαφημίσεις, ώστε η εφαρμογή να αισθάνεται ολοκληρωμένη την πρώτη μέρα αντί να ζητά από τους βασικούς χρήστες να συναρμολογήσουν μόνοι τους ολόκληρη τη στοίβα.

Η διαφορά είναι ότι η χρεοκοπία δεν πρέπει να γίνει η φυλακή. Ένας χρήστης θα πρέπει να μπορεί να αλλάζει RPC, ροές δεδομένων, παρουσίες, συστήματα κατάταξης, διαφημίσεις και παρόχους εύρεσης ή να καταργεί εντελώς την κατάταξη. Η εφαρμογή μπορεί να εκτιμηθεί κατά την πρώτη κυκλοφορία, ενώ διατηρεί κάθε σημαντική υπηρεσία αντικαταστάσιμη.

Αυτό κάνει την εφαρμογή πιο προσαρμόσιμη από μια συμβατική πλατφόρμα. Ένας χρήστης μπορεί να διατηρήσει την προεπιλεγμένη κατάταξη ροή με διαφημίσεις. Κάποιος άλλος μπορεί να χρησιμοποιήσει μια χρονολογική ροή χωρίς κατάταξη. Ένας άλλος μπορεί να επιλέξει ένα RPC με επίκεντρο το απόρρητο, μια υπηρεσία εντοπισμού που εκτελείται από την κοινότητα, μια πληρωμένη ροή χωρίς διαφημίσεις ή έναν εξειδικευμένο αλγόριθμο που έχει δημιουργηθεί για μια συγκεκριμένη υποκουλτούρα.

## Κοινότητες μεταξύ πελατών

Οι κοινότητες θα πρέπει να είναι πολύ πιο σημαντικές από μεμονωμένες ομάδες μέσα σε μία εφαρμογή.

Στο X/Twitter, οι κοινότητες περιορίζονται στο X. Μπορεί να είναι χρήσιμες, αλλά κληρονομούν τα όρια μιας πλατφόρμας, ενός συστήματος λογαριασμού, μιας στοίβας προτάσεων και μιας επιφάνειας προϊόντος.

Μια κοινότητα Bitsocial μπορεί να δημιουργηθεί, να φιλοξενηθεί, να ανακαλυφθεί και να χρησιμοποιηθεί μέσω διαφορετικών πελατών. Αυτό σημαίνει ότι η ναυαρχίδα εφαρμογή μπορεί να εμφανίζει κοινότητες και αναρτήσεις από το ευρύτερο δίκτυο Bitsocial, όχι μόνο από χρήστες που ξεκίνησαν μέσα στην εμβληματική εφαρμογή. Μια κοινότητα θα μπορούσε να έχει δραστηριότητα από έναν πελάτη εικόνας πίνακα, έναν πελάτη συζήτησης τύπου Reddit, έναν εξειδικευμένο πελάτη φόρουμ, μια εφαρμογή για κινητά και την εφαρμογή ναυαρχίδα ταυτόχρονα.

Αυτό είναι το βασικό πλεονέκτημα του αποτελέσματος δικτύου: ένας πελάτης μπορεί να αισθάνεται οικείος στους κύριους χρήστες, ενώ εξακολουθεί να αντλεί αξία από πολλούς πελάτες, κόμβους κοινότητας, παρόχους RPC και ανεξάρτητες υπηρεσίες.

## Προαιρετικοί αλγόριθμοι τροφοδοσίας

Η εμβληματική εφαρμογή δεν θα πρέπει να επιβάλλει σε όλους ένα παγκόσμιο σύστημα κατάταξης.

Οι αλγόριθμοι ροής θα πρέπει να είναι ενεργοποιημένοι. Ένας χρήστης θα μπορούσε να επιλέξει έναν αλγόριθμο από μια αγορά, να αλλάξει παρόχους, να χρησιμοποιήσει έναν αλγόριθμο από μια εταιρεία, να χρησιμοποιήσει έναν αλγόριθμο που εκτελείται από έναν ανώνυμο χειριστή, να χρησιμοποιήσει έναν που δημιουργήθηκε από μια κοινότητα, να εκτελέσει έναν προσωπικό ή να μην χρησιμοποιήσει καθόλου αλγόριθμο.

Οι δημόσιοι πάροχοι RPC είναι ένα φυσικό μέρος για να ανταγωνίζονται αυτές οι υπηρεσίες. Μπορούν να ευρετηριάσουν, να ταξινομήσουν και να προτείνουν περιεχόμενο, αλλά δεν θα πρέπει να είναι κάτοχοι του χρήστη ή του προφίλ.

Αυτές οι υπηρεσίες μπορούν επίσης να ανταγωνιστούν στο σχήμα της ίδιας της εφαρμογής. Ένα RPC μπορεί να παρέχει μια ταξινομημένη ροή με διαφημίσεις. Κάποιος άλλος μπορεί να παρέχει μια μη ταξινομημένη χρονολογική ροή. Κάποιος άλλος μπορεί να ειδικεύεται στο απόρρητο, τη μετάφραση, τη μετριοπάθεια, την ανακάλυψη κοινότητας ή ένα εξειδικευμένο κοινωνικό γράφημα.

Εάν τα οικονομικά λειτουργούν, οι υπηρεσίες τροφοδοσίας που υποστηρίζονται από RPC θα μπορούσαν να προσθέσουν λειτουργίες AI παρόμοιες με αυτές που προσπαθούν να βάλουν οι κύριες πλατφόρμες στις ροές τους: αυτόματες μεταφράσεις, περιλήψεις, απαντήσεις που υποβοηθούνται από bot, απαντήσεις αναζήτησης, βοήθεια εποπτείας ή περιβάλλον στυλ σημειώσεων κοινότητας.

Αυτά τα χαρακτηριστικά θα πρέπει να είναι επιλογές υπηρεσιών και όχι απαιτήσεις πρωτοκόλλου. Ένα προεπιλεγμένο RPC μπορεί να ανταγωνιστεί προσφέροντας μια πιο πλούσια ροή, αλλά οι χρήστες και οι ανταγωνιστικοί πελάτες θα πρέπει να μπορούν να επιλέγουν απλούστερες, ιδιωτικές, χρονολογικές, χωρίς διαφημίσεις ή συγκεκριμένες εναλλακτικές λύσεις για την κοινότητα.

## RPC μη στερητική της ελευθερίας

Κάθε χρήστης θα πρέπει να μπορεί να συμμετέχει ως πλήρης κόμβος peer-to-peer μέσω του RPC χωρίς να εκχωρεί στον πάροχο RPC την κυριότητα της ταυτότητας ή του προφίλ του.

Η φιλοξενούμενη διαδρομή έχει σημασία επειδή οι περισσότεροι χρήστες δεν θα ξεκινήσουν εκτελώντας έναν διακομιστή. Η διαδρομή εξόδου είναι εξίσου σημαντική: ένας χρήστης θα πρέπει να μπορεί να μετακινείται στον δικό του κόμβο προφίλ σε υλικό χαμηλών προδιαγραφών, συμπεριλαμβανομένου ενός Raspberry Pi, όποτε θέλει.

Αυτή είναι η διαφορά μεταξύ ευκολίας και φύλαξης.

## Γιατί μπορεί να γίνει εφαρμογή για τα πάντα

Εάν το Bitsocial Chain δίνει στις εφαρμογές ανθεκτικές ονομασίες, πληρωμές, φιλοδωρήματα, βραβεία και άλλες οικονομικές ράγες, η ναυαρχίδα της εφαρμογής θα μπορούσε να γίνει κάτι πολύ περισσότερο από ένα πρόγραμμα-πελάτη τροφοδοσίας.

Ο σημαντικός περιορισμός είναι ότι η εφαρμογή δεν πρέπει να γίνει ο νέος κάτοχος του δικτύου. Μπορεί να είναι ένας μεγάλος πελάτης, ίσως ακόμη και ο πιο δημοφιλής πελάτης, ενώ εξακολουθεί να αφήνει χώρο για ανταγωνιστικές εφαρμογές, ανταγωνιστικά RPC, ανταγωνιστικούς αλγόριθμους τροφοδοσίας και αυτο-φιλοξενούμενους κόμβους προφίλ.
