# Προσαρμοσμένες προκλήσεις Anti-Spam

Το Bitsocial δεν προϋποθέτει μια καθολική προστασία ανεπιθύμητων μηνυμάτων. Κάθε κοινότητα μπορεί να αποφασίσει τι απαιτεί μια έγκυρη ανάρτηση ή απάντηση, και αυτή η πρόκληση μπορεί να είναι τόσο ελαφριά ή με γνώμονα όσο θέλει ο χειριστής.

Για την πλήρη ροή στα Αγγλικά από τις διευθύνσεις της κοινότητας έως την ανάρτηση με περιορισμένη πρόκληση, διαβάστε [Μια πλήρης απλή εξήγηση του πρωτοκόλλου Bitsocial](./layman-protocol-explanation.md).

## Τι πρόκληση μπορεί να είναι

- Ένα captcha
- Έλεγχος ηλικίας λογαριασμού ή φήμης
- Επαλήθευση SMS
- Μια μικρή πληρωμή
- Απαίτηση διακριτικού ή NFT
- Κανόνες βασισμένοι σε IP
- Μια λίστα επιτρεπόμενων
- Οποιαδήποτε άλλη πολιτική που μπορεί να εκφραστεί σε κώδικα

## Γιατί αυτό έχει σημασία

Η αντίσταση στα ανεπιθύμητα μηνύματα παραμένει τοπική στην κοινότητα αντί να ανατίθεται σε τρίτους σε μια αρχή εποπτείας σε όλο το πρωτόκολλο. Ο κόμβος που φιλοξενεί την κοινότητα μπορεί να επικοινωνήσει την πρόκληση peer-to-peer, πράγμα που σημαίνει ότι οι κοινότητες μπορούν να προσαρμόσουν τις άμυνές τους χωρίς να εξαναγκάσουν ολόκληρο το δίκτυο σε μία πολιτική.

## Τι αλλάζει αυτό

Σε μια κεντρική πλατφόρμα, η πολιτική κατά των ανεπιθύμητων μηνυμάτων συνήθως γίνεται πολιτική προϊόντων. Όταν συμβεί αυτό, μια εταιρεία αποφασίζει τι θεωρείται έγκυρος λογαριασμός, έγκυρη ανάρτηση ή αποδεκτή προσέγγιση χρηστών.

Το Bitsocial διακόπτει αυτή τη σύζευξη:

- το πρωτόκολλο παραμένει ουδέτερο
- η κοινότητα επιλέγει την πρόκληση
- Οι εφαρμογές αποφασίζουν πώς να παρουσιάσουν ή να αφαιρέσουν αυτήν την πρόκληση

Αυτό ταιριάζει καλύτερα σε ένα δίκτυο που προορίζεται να φιλοξενεί πολλά είδη κοινωνικών προϊόντων παρά μια παγκόσμια προεπιλογή.
