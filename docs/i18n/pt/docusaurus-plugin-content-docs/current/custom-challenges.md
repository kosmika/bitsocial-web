# Desafios antispam personalizados

O Bitsocial não assume uma defesa universal contra spam. Cada comunidade pode decidir o que uma postagem ou resposta válida exige, e esse desafio pode ser tão leve ou opinativo quanto o operador desejar.

Para obter o fluxo completo em inglês simples, desde endereços da comunidade até postagens restritas a desafios, leia [Uma explicação completa para leigos do protocolo Bitsocial](./layman-protocol-explanation.md).

## Que desafio pode ser

- Um captcha
- Uma verificação de idade da conta ou reputação
- Verificação por SMS
- Um pequeno pagamento
- Um token ou requisito NFT
- Regras baseadas em IP
- Uma lista de permissões
- Qualquer outra política que possa ser expressa em código

## Por que isso é importante

A resistência ao spam permanece local na comunidade, em vez de ser terceirizada para uma autoridade de moderação de todo o protocolo. O nó que hospeda a comunidade pode comunicar o desafio ponto a ponto, o que significa que as comunidades podem adaptar as suas defesas sem forçar toda a rede a uma política.

## O que isso muda

Numa plataforma centralizada, a política anti-spam geralmente se torna uma política de produto. Quando isso acontece, uma empresa decide o que conta como uma conta válida, uma postagem válida ou um alcance aceitável.

Bitsocial quebra esse acoplamento:

- o protocolo permanece neutro
- a comunidade escolhe o desafio
- os aplicativos decidem como apresentar ou abstrair esse desafio

Isto é mais adequado para uma rede destinada a hospedar muitos tipos de produtos sociais, em vez de um padrão global.
