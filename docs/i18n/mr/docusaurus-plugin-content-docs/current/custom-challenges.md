# सानुकूल अँटी-स्पॅम आव्हाने

Bitsocial एक सार्वत्रिक स्पॅम संरक्षण गृहीत धरत नाही. प्रत्येक समुदाय हे ठरवू शकतो की वैध पोस्ट किंवा प्रत्युत्तराची आवश्यकता काय आहे आणि ते आव्हान ऑपरेटरला हवे तितके हलके किंवा मतप्रदर्शन असू शकते.

सामुदायिक पत्त्यांपासून आव्हान-गेट पोस्टिंगपर्यंतच्या संपूर्ण साध्या-इंग्रजी प्रवाहासाठी, वाचा [बिटसोशियल प्रोटोकॉलचे संपूर्ण सामान्य स्पष्टीकरण](./layman-protocol-explanation.md).

## आव्हान काय असू शकते

- कॅप्चा
- खाते-वय किंवा प्रतिष्ठा तपासणी
- एसएमएस पडताळणी
- एक लहान पेमेंट
- टोकन किंवा NFT आवश्यकता
- IP-आधारित नियम
- अनुमत सूची
- कोडमध्ये व्यक्त करता येणारे कोणतेही अन्य धोरण

## हे महत्त्वाचे का आहे

स्पॅम प्रतिकार प्रोटोकॉल-व्यापी नियंत्रण प्राधिकरणाकडे आउटसोर्स करण्याऐवजी समुदायासाठी स्थानिक राहतो. समुदायाला होस्ट करणारा नोड पीअर-टू-पीअर आव्हानाशी संवाद साधू शकतो, याचा अर्थ समुदाय संपूर्ण नेटवर्कला एका धोरणात न लावता त्यांचे संरक्षण स्वीकारू शकतात.

## हे काय बदलते

केंद्रीकृत प्लॅटफॉर्मवर, स्पॅम विरोधी धोरण सहसा उत्पादन धोरण बनते. जेव्हा असे होते, तेव्हा एक कंपनी ठरवते की वैध खाते, वैध पोस्ट किंवा स्वीकार्य पोहोच म्हणून काय मोजले जाते.

बिटसोशल त्या कपलिंगला ब्रेक करते:

- प्रोटोकॉल तटस्थ राहतो
- समुदाय आव्हान निवडतो
- ॲप्स ते आव्हान कसे सादर करायचे किंवा त्याचे सार कसे काढायचे हे ठरवतात

एका जागतिक डीफाल्टऐवजी अनेक प्रकारच्या सामाजिक उत्पादनांचे आयोजन करण्यासाठी हे नेटवर्कसाठी अधिक योग्य आहे.
